Project

Priority-Conditioned Retention

기말 프로젝트. 상태 공간 모델(Mamba)은 과거 전체를 상수 크기 hidden state 하나에 압축해 선형 추론을 얻지만, 그 대가로 정확한 정보 복원(recall)에서 Transformer에 크게 뒤진다. 우리는 Transformer 진영의 importance-aware KV-캐시 압축 원리를 연속-상태 SSM으로 일반화한다.

핵심 통찰 — 이산↔연속 통합 축

importance-aware 압축의 본질은 한 줄이다: 중요한 정보일수록 더 많은 비트(정밀도/슬롯)를 차지한다. 이산 캐시는 "토큰당 비트"를 할당하지만, Mamba에는 슬롯이 없다. 우리는 보존 정보량 rᵢ = f(importanceᵢ) 를 공통 축으로 두고, Mamba를 per-item precision allocation의 연속 극한으로 재정식화한다.

01 · Problem

두 가지 recall 실패 원인 분해

(a) 무엇을 보존/망각할지 못 고르는 Selectivity 천장(고칠 수 있음)과 (b) 저장할 비트 자체가 부족한 Capacity 임계점(물리적 한계)을 분리한다.

02 · Method

데이터단 우선순위 문법

[P1]>[P2]>[P3] 태그로 Mamba의 암묵적 recency 압축 정책을 task-defined importance 정책으로 교체한다. 아키텍처는 무수정.

03 · Mechanism

Surviving Contribution

직접 측정량은 raw Δ가 아니라 게이트 곱 rᵢ = ∏ Āτ — 토큰 i의 기여가 쿼리 지점까지 얼마나 살아남는가 — 이다.

04 · Evaluation

Belady 대비 Retention Efficiency

오프라인 최적(Belady-MIN) 대비 비율로 측정하고, 전체 gap을 정책 최적성 gap + 연속-state 구현 gap 2단으로 분해한다.

K · b ≤ |h| ⇒ K* ≈ c · d_state (용량 임계점)

핵심 가설

  • H3우선순위 문법은 포화 시 [P1]을 보존하고 [P3]부터 보존도를 낮춘다.
  • H4surviving contribution rᵢ가 [P1] > 일반 > [P3] 로 차등(메커니즘 증거).
  • H5[P3]는 저부하에선 회상되고 포화 이후에만 탈락한다 (eviction-under-pressure).
  • H6이산 캐시의 비트폭을 정하는 동일한 importance 신호가 Mamba의 유효 보존량을 예측한다.

로드맵

Tier 0 8주 (과제) — priority-coupled MQAR + Belady 기준선 → 분기 곡선·Δ probe.
Tier 1 용량 배분 법칙 + 간섭(interference) 프런티어.
Tier 2 사전학습 Mamba-2 실모델 전이 — 프롬프트 우선순위 포맷팅 → recall↑.
Tier 3 activation patching 인과 검증 + priority-weighted bound 이론.
